Select the font you'd like to use

For now only fonts in the following formats are supported: TrueType (TTF), OpenType (OTF) and Printer Font Binary (PFB).

See Fonts and the Law at fontembedding.com for more information. Fonts produced by the following foundries/vendors/creators are known to be safe: Adobe Systems. The following are known to require separate or extended licenses for Web Embedding: ITC (separate), Linotype (extended), FontFont (separate).

Include the following glyphs (if available)

Includes all available glyphs. Highly unrecommended.
Basic Latin uppercase letters (A-Z). (26 glyphs)
Basic Latin lowercase letters (a-z). (26 glyphs)
Basic Latin digits (0-9). (10 glyphs)
Basic Latin punctuation (!@#%...). (33 glyphs)
Basic Latin glyphs within the Unicode range 0x0020 to 0x007E. (95 glyphs)
Latin glyphs within the Unicode range 0x00A0 to 0x00FF. Includes punctuation, currency symbols, numerals and common accented characters. (96 glyphs)
Extended Latin glyphs within the Unicode range 0x0100 to 0x017F. (128 glyphs)
Extended Latin glyphs within the Unicode range 0x0180 to 0x024F. (208 glyphs)
Combined glyphs, such as ffi and ch. For performance reasons this feature is currently unsupported.

Depending on the font you're using, some glyphs might not be selected correctly. This is a known bug and we are working on it. So far, we've only had problems with "-" (hyphen) and " " (space).

Security

(example.org, www.example.org)

Separate multiple domains by either commas or spaces. Please make sure to include your domain both with and without «www» if necessary (if you're not sure, it's probably necessary). Please note that is not a fool-proof solution but will prevent people from hotlinking your fonts.

Performance & file size

A lower value (64 at minimum) results in slightly better performance and smaller file size at the cost of accuracy. Values lower than 256 are not recommended unless you are OK with visible loss of detail. The native values in quality fonts are usually either 1000 or 2048.

units

You should keep this value fairly low, at about 1% of the em-size of the font. Optimizations are performed after the font has been scaled (if at all).

Also, try to make sure your server serves gzipped JavaScript. 70-80% drops in file size are not unheard of.
